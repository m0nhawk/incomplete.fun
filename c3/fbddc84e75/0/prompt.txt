You are helping ship changes by committing and pushing.

Steps:
1. Follow the `commit` skill workflow:
   - inspect status and diffs
   - stage appropriate files
   - create a concise commit
2. Then follow the `push` skill workflow:
   - identify current branch
   - push to the configured upstream
   - if needed, set upstream with `git push -u origin <current-branch>`
3. Report:
   - commit hash
   - commit message
   - branch pushed
